## Problem

resume-panel 실사용 중 인터뷰 흐름과 조사 파이프라인에서 다음 문제가 반복적으로 드러났다.

- `이력서 없음` 경로에서 직군 질문이 개발 직군 위주로 수렴해 비개발 직군 사용자를 배제하는 인상을 준다.
- 회사/기간 입력 후 리서처 결과를 기다리는 동안 2분 가까이 멈춘 것처럼 보여 흐름이 끊긴다.
- 회사 리서치는 병렬 풀스캔 + Playwright fallback + 재주입 구조 때문에 토큰 소모가 과도하다.
- AskUserQuestion 셀렉트 박스는 빈 영역 `Enter`로 종료될 수 있고, 이후 텍스트 폴백으로만 내려가 복구 UX가 약하다.
- 질문 UI는 단일 선택 중심이라 멀티 셀렉트, 여러 질문 배치, 마지막 확인 단계 같은 실제 입력 패턴을 지원하지 못한다.
- 리서처 결과가 메인 세션에서 영속적으로 모이지 않아, 유의미한 조사 결과가 나와도 점진적으로 활용하기 어렵다.
- 서브에이전트 호출 시 최소 컨텍스트 전략이 부족해 불필요한 노이즈와 토큰 낭비 우려가 있다.

## Solution

우선순위를 나눠 다음 방향으로 개선한다.

- 셀렉트 박스 안정화
  빈 영역 `Enter` 무효화, 비정상 종료 시 동일 질문 재오픈, 텍스트 폴백 조건 축소.
- 리서치 대기 시간 단축
  리서처는 백그라운드에서 돌리고, 앞단에서는 직군/타겟 포지션/강조 회사/현재 재직 여부 같은 비의존 질문을 계속 진행.
- 리서처 경량화
  과거 회사는 얕게, 타겟 회사/JD는 깊게 조사하고, 질문 생성에 필요한 최소 팩트만 먼저 확보.
- 리서처 결과 캐시 도입
  `.resume-panel/research-cache.json` 또는 `research-inbox.jsonl`을 두고, 회사별/타겟 JD별 조사 결과를 유의미한 단위로 즉시 저장.
- 점진적 프로파일링
  프로파일러는 조사 완료 후 1회 분석이 아니라, 신규 조사 결과가 들어올 때마다 증분 분석.
- 최소 컨텍스트 서브에이전트 패턴 정리
  메인 오케스트레이터는 서브에이전트에 작업 범위, 필요한 파일, 압축 요약만 전달.
- 질문 UI 확장 검토
  멀티 셀렉트 체크박스, 여러 질문 묶음, 마지막 확인 단계 같은 입력 UX를 설계.
