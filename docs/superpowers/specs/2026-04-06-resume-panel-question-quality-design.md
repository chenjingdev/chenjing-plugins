# Resume Panel 질문 품질 개선 설계

## 배경

resume-panel의 에이전트 질문 시스템에 세 가지 문제가 있다:

1. 유저가 선택지에 없는 답을 하고 싶을 때 "직접 입력" 옵션이 없음
2. 에이전트가 대화 맥락을 충분히 반영하지 못하고 리서처 데이터에서 기계적으로 질문을 뽑음
3. 에이전트마다 캐릭터가 다른데 질문 포맷/톤이 전부 동일함

## 접근법

오케스트레이터(SKILL.md) + 에이전트 프롬프트를 함께 수정한다. 에이전트한테 "잘 해라"고 규칙만 추가하는 게 아니라, 오케스트레이터가 좋은 재료(대화 브리핑)를 넘겨서 질문 품질을 구조적으로 올린다.

## A. "직접입력" 선택지 추가

### 규칙

- 모든 에이전트의 선택지 마지막에 `직접입력` 항목을 추가한다
- 표현은 통일: `N) 직접입력`
- 유저가 이 항목을 고르면 자유 텍스트 입력으로 처리한다

### 산출 형식 변경

변경 전:
```
[에이전트명] {질문 텍스트}
  1) {선택지1}
  2) {선택지2}
  3) {선택지3 (선택)}
```

변경 후:
```
[에이전트명] {질문 텍스트}
  1) {선택지1}
  2) {선택지2}
  3) {선택지3 (선택)}
  4) 직접입력
```

### 수정 대상

- `plugins/resume/.claude/agents/senior-dev.md`
- `plugins/resume/.claude/agents/cto.md`
- `plugins/resume/.claude/agents/recruiter.md`
- `plugins/resume/.claude/agents/hr.md`
- `plugins/resume/.claude/agents/coffee-chat/*.md` (5개 전부)
- `plugins/resume/skills/resume-panel/SKILL.md` (허용 질문 예시, 산출 형식 부분)

## B. 오케스트레이터 대화 브리핑

### 변경 내용

SKILL.md의 "에이전트 호출 방법" 섹션에서, 오케스트레이터가 에이전트 호출 시 넘기는 컨텍스트에 **대화 브리핑**을 추가한다.

### 대화 브리핑 형식

```
## 대화 브리핑
- 유저가 지금까지 강조한 키워드/주제: [예: 자동화, 파이프라인, 시간 절감]
- 이미 다룬 영역: [예: CI/CD 구축 경험, 팀 규모]
- 아직 안 다룬 영역: [예: 비즈니스 임팩트 수치, 장애 대응]
- 유저의 직전 답변 요약: [1-2문장]
```

### 오케스트레이터 규칙 추가

SKILL.md "에이전트 호출 방법" 섹션에 추가:

> 에이전트를 호출하기 전에 대화 브리핑을 정리한다. 유저가 지금까지 강조한 키워드, 이미 다룬 영역, 아직 안 다룬 영역, 직전 답변 요약을 구조화해서 에이전트에게 전달한다.

### 에이전트 프롬프트 규칙 추가

모든 프론트스테이지 에이전트 프롬프트의 "질문 생성 규칙 > 반드시 지킬 것"에 추가:

> 대화 브리핑의 '이미 다룬 영역'을 다시 묻지 않는다. '아직 안 다룬 영역' 중에서 질문을 생성한다. 유저가 강조한 키워드가 있으면 그것과 연결되는 질문을 우선한다.

### 수정 대상

- `plugins/resume/skills/resume-panel/SKILL.md` (에이전트 호출 방법 섹션)
- `plugins/resume/.claude/agents/senior-dev.md`
- `plugins/resume/.claude/agents/cto.md`
- `plugins/resume/.claude/agents/recruiter.md`
- `plugins/resume/.claude/agents/hr.md`
- `plugins/resume/.claude/agents/coffee-chat/*.md` (5개 전부)

## C. 에이전트별 질문 톤/포맷 차별화

### 변경 내용

각 에이전트 프롬프트의 질문 패턴 예시와 산출 형식을 에이전트 캐릭터에 맞게 분리한다.

### 시니어 개발자 — 동료 톤, 기술 용어 편하게

```
{기술A} 도입한 거 봤는데, 이거 직접 밀었어 아니면 이미 있던 거야?
1) 내가 제안해서 도입
2) 기존에 있었고 활용만
3) 마이그레이션 작업
4) 직접입력
```

톤 가이드: 옆자리 동료한테 묻듯이. "~인데", "~했어?", "~거야?" 체. 기술 용어는 설명 없이 바로 씀.

### CTO — 전략적, 숫자/임팩트 중심

```
{서비스} MAU {N}만인데, 네가 한 {작업}이 비즈니스 지표에 얼마나 찍혔어?
1) 전환율/이탈률 개선 — 수치 있음
2) 비용/시간 절감 — 수치 있음
3) 체감은 있었는데 측정은 안 했음
4) 직접입력
```

톤 가이드: 한 발 떨어져서 전략적으로. 항상 숫자/스케일을 묻는다. "~찍혔어?", "~영향이 있었어?", "~규모였어?" 체.

### 채용담당자 — 냉정, 직설적, JD 기준

```
JD에 '{요구사항}' 필수인데 관련 에피소드가 아직 없어.
1) 있는데 아직 안 말한 거
2) 진짜 없음
3) 직접입력
```

톤 가이드: 사실 기반으로 끊어서 판단. 위로 없음. "~없어.", "~어려움.", "~어느 쪽?" 체.

### 인사담당자 — 조직/사람 관점, 구체적 상황 세팅

```
{팀} {N}명에서 주니어가 들어왔을 때 네 역할이 뭐였어?
1) 온보딩 직접 담당
2) 코드리뷰로 가이드
3) 직접 관여 안 했음
4) 직접입력
```

톤 가이드: 상황을 먼저 세팅하고 역할을 묻는다. "~일 때", "~상황에서", "~뭐였어?" 체.

### 커피챗봇 — 각 페르소나 유지

기존 페르소나별 캐릭터를 유지하되, 동일하게 `직접입력` 선택지와 대화 브리핑 규칙을 추가한다.

### 수정 대상

- `plugins/resume/.claude/agents/senior-dev.md` (질문 패턴 + 톤 가이드)
- `plugins/resume/.claude/agents/cto.md` (질문 패턴 + 톤 가이드)
- `plugins/resume/.claude/agents/recruiter.md` (질문 패턴 + 톤 가이드)
- `plugins/resume/.claude/agents/hr.md` (질문 패턴 + 톤 가이드)
- `plugins/resume/.claude/agents/coffee-chat/*.md` (직접입력 + 대화 브리핑만)
