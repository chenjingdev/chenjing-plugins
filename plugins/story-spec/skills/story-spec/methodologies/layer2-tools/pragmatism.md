# Pragmatism (실용주의 격률)

## 무엇인가

미국 철학자 Charles S. Peirce가 1878년 논문 *How to Make Our Ideas Clear*에서 정식화하고, William James가 1898-1907년 강연·저서 *Pragmatism*에서 대중화한 의미 검증 격률. 한 명제가 *참이라면 실제로 어떤 차이가 발생하는가*를 묻는다. 차이가 없으면 그 명제는 의미가 없다(Peirce의 격률), 또는 차이만이 의미다(James의 강한 형태).

story-spec에서는 추상적 비전("앱이 잘 작동한다"·"사용자가 만족한다")을 *관찰 가능한 차이*로 강제 변환하는 도구로 쓴다. L6 codespeak spec의 acceptance criteria 1순위, L4의 success condition, L1의 끝 보조다.

## 어울리는 LOD

- **L6 codespeak spec — 1순위 ⭐** (acceptance criteria를 *어떤 행동의 변화로 관찰 가능한가*로 강제 정의)
- L4 시나리오 카드 보조 (success condition 필드)
- L1 끝 보조 (Pixar Pitch의 7번째 문장을 관찰 가능 형태로)

## 어울리는 카테고리 (boost)

- **tool-productivity** — 도구 앱은 "효율 향상" 같은 추상이 흔해서 격률의 강제 변환이 결정적

universal: 모든 카테고리에서 작동한다. penalty 없음. 다만 game-habit·content-media에서는 *차이*가 정서·취향이라 측정 가능 형태가 약간 거칠어질 수 있어 Miracle Question·Projective와 짝으로 쓰인다.

## 핵심 질문 패턴

1. "이것이 참이라면 *실제로 어떤 차이*가 발생해요? 가장 작고 관찰 가능한 차이부터요."
2. "이 기능이 있을 때와 없을 때, 사용자의 화요일 오후가 어떻게 달라져요? 한 동작 단위로요."
3. "성공했다는 걸 무엇으로 알아요? *누가, 언제, 어떤 행동의 변화*로요?"
4. "그 차이가 일어났는지 일어나지 않았는지를 *외부에서 보면* 어떻게 구분해요?"
5. "그 비전이 거짓이라고 가정해보세요. 거짓일 때와 참일 때, 사용자의 행동에서 무엇이 달라야 해요?"

## 안티패턴

- **추상의 추상으로 답함** — "더 좋은 경험" / "사용자 만족도 향상" 같은 답은 차이 정의가 아니라 차이의 *라벨*이다. "그 *더 좋은 경험*을 누가 어떻게 알아채요?"로 한 번 더 내린다
- **숫자 KPI로 곧장 환원** — "전환율 +5%"는 측정 가능하지만 *어떤 사용자 행동이 달라져서* 그 숫자가 움직이는지를 건너뛴다. 행동 변화가 먼저, 지표는 그 행동의 그림자
- **양자택일 가능 여부를 안 묻고 끝냄** — 격률의 핵심은 "참일 때와 거짓일 때 *구분 가능한가*". 구분이 안 되면 명제가 공허하다는 신호. "거짓이어도 같은 풍경이라면 이 비전은 빠진다"가 결정 룰
- **현재의 추세를 미래 차이로 위장** — "사용자가 더 자주 쓸 것이다"는 추세 진술. 격률은 "더 자주 쓴다는 게 *몇 번에서 몇 번으로*, *어떤 맥락에서*"를 강제한다

## Layer 1 메타 자세와의 결합 메모

- **Clean Language** — "차이"를 묻되 사용자의 단어로 묻는다. 사용자가 "*숨통이 트인다*"고 했다면 "효율이 오른다"로 바꾸지 않고 "그 *숨통이 트이는* 순간을 외부에서 보면 무엇이 달라져 있나요?"로 격률을 적용
- **Miracle Question과 짝** — Miracle이 *기적의 풍경*을 끌어낸다면, Pragmatism은 그 풍경의 *각 요소를 외부 관찰 가능 형태로* 정밀화한다. "햇빛이 든 부엌에서 커피를 내린다" → "알람을 끄지 않고 일어나는 비율이 일주일에 며칠인가"
- **OARS의 Reflect** — 사용자가 던진 차이를 "그러니까 *[사용자 단어]*가 일어나면 *[관찰 가능 행동]*이 보이는 거네요. 맞나요?"로 되돌려 자기 정정 트리거
- **Bloom 게이지** — Pragmatism은 사용자를 "이해/적용" 층위에서 "분석/평가" 층위로 끌어올리는 도구. "이 비전을 거짓이라 가정하면 무엇이 달라져요?"는 평가 층위 질문
- **System 1/2 이중 캡처** — "처음 떠오른 차이"(System 1)와 "30초 더 정교화한 차이"(System 2)를 둘 다 적는다. 보통 System 1은 라벨에 가깝고 System 2는 행동 단위에 가까워 격률이 자연히 적용된다
