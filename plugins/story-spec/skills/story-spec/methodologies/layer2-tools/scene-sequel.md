# Scene-Sequel

## 무엇인가

Dwight Swain이 1965년 저서 *Techniques of the Selling Writer*에서 정립한 소설 구조 분석 단위. 스토리는 두 종류의 단위가 번갈아 진행한다.

- **Scene** = *목표 → 갈등 → 결말*. 캐릭터가 명시적 목표를 가지고 행동하다 무엇에 막혀, 어떤 결말(보통 좌절·반전·부분 성공)에 도달한다
- **Sequel** = *반응 → 딜레마 → 결정*. 직전 Scene 직후, 캐릭터가 내적으로 반응하고, 어떤 선택지 사이에서 헤매고, 다음 행동을 결정한다

story-spec에서는 L5 장면 LOD의 2순위 도구로 쓴다. 한 시나리오 카드(L4)에 대해 *외적 행동의 단위*인 Scene과 *내적 처리의 단위*인 Sequel을 분리해 적도록 강제한다. 단순 장면 묘사가 놓치는 "사용자가 그 직후에 무엇을 느끼고 어떤 선택지 사이에 있었나"를 끌어낸다.

## 어울리는 LOD

- **L5 장면 — 2순위** (Master/Apprentice가 *행동의 미시 흐름*을 채집했다면, Scene-Sequel은 *행동과 그 직후 내적 처리의 쌍*을 강제로 분리)
- L4 시나리오 카드 보조 (카드 자체를 Scene 단위로 잘라낼 때)

## 어울리는 카테고리 (boost)

- **marketplace-transaction** — 거래 앱은 결제·매칭 직후 *후회/안도/재선택* 같은 Sequel이 결정적
- **game-habit** — 한 세션의 클리어·실패 직후 사용자가 어떤 선택지 사이를 오가는지가 리텐션 핵심

penalty 없음. 단 tool-productivity는 Sequel이 짧고 압축돼 Master/Apprentice가 더 잘 맞는 경우가 많다.

## 핵심 질문 패턴

**Scene 쪽 (외적 단위):**
1. "이 장면에서 사용자의 *목표*는 무엇이에요? 한 문장으로요."
2. "그 목표를 *방해*하는 건 무엇이에요? 시스템이에요, 다른 사용자에요, 자기 자신이에요?"
3. "이 장면이 어떤 *결말*로 닫혀요? 성공·실패·반전·보류 중 어느 쪽에 가까워요?"

**Sequel 쪽 (직후 내적 단위):**
4. "그 장면 직후, 사용자는 어떤 *반응*을 느껴요? 몸의 어디에서 그게 일어나요?"
5. "그때 사용자가 *헤매는 선택지* 두세 개는 무엇이에요? 'A를 다시 할까, B로 갈까, 그냥 닫아버릴까' 같은 형태로요."
6. "결국 무엇을 *결정*하나요? 그 결정을 끌어낸 결정타는 무엇이에요?"

## 안티패턴

- **Scene만 적고 Sequel을 빠뜨림** — 가장 흔한 위반. "결제가 완료됐다"로 끝내고 직후의 안도/불안/후회를 묻지 않으면, 다음 세션 진입 동기를 잃는다. 모든 Scene 카드 옆에 Sequel 칸을 강제 배치한다
- **Sequel을 *다음 Scene의 동기*로만 환원** — "그래서 다음에 X를 누른다"로만 적으면 내적 처리가 사라진다. *반응-딜레마-결정* 세 칸을 다 채운다
- **목표·갈등·결말을 한 덩어리로 뭉침** — "이 화면에서 사용자가 결제하는 장면"은 Scene의 라벨이지 분해가 아니다. 셋을 분리해 적는 것 자체가 산출물
- **딜레마를 합리적 선택지로만 채움** — "A안과 B안 중 비교"는 분석 위장. 실제 사용자의 딜레마는 "다시 시도할까 / 그냥 닫고 나중에 / 친구한테 물어볼까" 같은 *행동 후보들*이다

## Layer 1 메타 자세와의 결합 메모

- **Clean Language** — Scene의 *결말*과 Sequel의 *반응*은 사용자가 직접 단어를 골라 부르는 곳이라 메타포 보존이 결정적. 사용자가 "*막힌 느낌*"이라고 하면 "장애감"·"좌절"로 바꾸지 않고 "*막힌 느낌*"을 그대로 산출물에 적는다
- **OARS의 Summarize** — 한 Scene-Sequel 쌍이 끝나면 "*[사용자 단어 목표]*를 하려다 *[사용자 단어 갈등]*에 막혀 *[사용자 단어 결말]*로 끝났고, 그 직후 *[사용자 단어 반응]*을 느꼈다" 한 줄로 압축해 되돌려준다
- **MICE Quotient(L2)와 결합** — 사용자가 큰 서사를 Inquiry축이라고 정한 상태라면, Scene의 *목표*는 "알고 싶은 것을 알아내는 것"으로 일관되게 잡힌다. L2의 축이 L5 Scene의 목표 슬롯을 안내한다
- **Yes-And** — Sequel의 딜레마 칸에서 "그건 비합리적인 선택지인데요"라고 잘라내면 사용자가 더 못 꺼낸다. "그 선택지도 있고요. 또 어떤 가능성이?"로 펼친다
- **System 1/2 이중 캡처** — Scene 직후 Sequel의 *반응*은 System 1(즉답), *결정*은 System 2(정교화)에 가깝다. 둘 다 따로 적는다
