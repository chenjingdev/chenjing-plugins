# MICE Quotient (MICE 지수)

## 무엇인가

Orson Scott Card가 *Characters and Viewpoint* (1988)에서 정리한 서사 분류 도구. 모든 이야기는 네 종류의 갈등 중 하나(또는 가중 조합)를 *주된 활(arc)*로 가진다는 관점:

- **M — Milieu (환경)**: 낯선 환경에 들어가 적응하고 떠나는 이야기. *반지의 제왕* 시작·끝.
- **I — Inquiry (수수께끼)**: 답해야 할 질문이 있고 그 답을 찾아내는 이야기. 추리물 일반.
- **C — Character (인물)**: 주인공의 정체성·역할·관계가 변하는 이야기. 성장 서사.
- **E — Event (외부 사건)**: 세계 질서가 깨졌고 그것을 회복(또는 새 질서 수립)하는 이야기. 재난·정치 드라마.

story-spec에서는 L2 Story Circle 8단계의 *주된 갈등 축*을 분류해, 어떤 단계에 인터뷰·산출물 비중을 둘지 결정하는 데 사용한다.

## 어울리는 LOD

- **L2 큰 서사 — 1순위 ⭐**
- L3 챕터의 "이 챕터의 갈등 축은 무엇인가" 분류 보조
- L4 시나리오 카드의 "장애물" 필드 분류(M·I·C·E 어디에서 오는 장애물인가)

## 어울리는 카테고리 (boost)

- **social-community** — 관계·정체성·소속이 본질이라 Character 축이 강함. MICE로 명확히 분류하면 인터뷰 비중이 잡힘
- **content-media** — 콘텐츠 소비는 보통 Milieu(새 콘텐츠 환경 적응) + Character(취향 변화)의 혼합. MICE로 비중을 정량화 가능
- **game-habit** — 게임·습관은 Character(자기 변화) 축이 가장 강한 분야

penalty: tool-productivity (도구 앱은 보통 Inquiry 단일 축이라 MICE 분류가 자명해 정보 가치 낮음 — 그 경우 Toyota Kata가 더 적합)

## 4축의 제품 적용 매핑

| MICE | 제품 적용 | Story Circle 강조점 | 시나리오 예 |
|---|---|---|---|
| **Milieu** | 낯선 도구·환경에 적응 | Go·Search 강조 | "Notion 처음 켰을 때부터 첫 페이지 만들기까지" |
| **Inquiry** | 정보·답·해결책 탐색 | Need·Search·Find 강조 | "내 문제가 무엇인지 이 앱으로 진단" |
| **Character** | 자기 정체성·습관 변화 | You·Need·Change 강조 | "운동 안 하던 사람이 매일 뛰는 사람으로" |
| **Event** | 외부 사건 대응·복구 | Need·Take·Return 강조 | "데이터 유실 → 복구 → 백업 습관 형성" |

## 핵심 질문 패턴

1. "이 사용자 여정의 *주된 갈등*은 다음 중 어느 쪽에 가까워요? — *(1) 환경에 적응하기* / *(2) 수수께끼나 답을 찾기* / *(3) 자신이 변하기* / *(4) 외부 사건에 대응하기*. 한 가지만 골라본다면요?"
2. "방금 1순위로 고른 게 [X]인데, 2순위로 깔리는 축은 무엇인가요? 두 축의 비중을 100점 만점으로 나눈다면 어떻게 나눌래요?"
3. "[1순위 축]이 가장 *강해지는 단계*는 Story Circle 8단계 중 어디인가요? 어디에서 시작해서 어디에서 절정에 이르나요?"
4. "이 [축]에서 사용자가 *떠안는 대가*(Take 단계)는 무엇인가요? 학습 비용·시간·관계 변화 어느 쪽이에요?"
5. "[축]을 잘못 분류하면 어떤 챕터에 비중을 잘못 둘 거 같아요? 한번 반대 축으로 가정해서 보면 어떤 차이가 생겨요?"

## Story Circle 단계별 강조 반영 룰

- **Milieu형으로 분류** → Go(앱 진입)·Search(탐색) 단계의 챕터를 더 자세히. 온보딩·empty state·discoverability에 시나리오 비중.
- **Inquiry형으로 분류** → Need(결핍 인식)·Find(발견) 단계 강조. 검색·필터·답안 제시 시나리오에 비중.
- **Character형으로 분류** → You(시작 자기)·Need·Change(변화한 자기) 강조. 사용자 성장·습관·정체성 마커에 비중.
- **Event형으로 분류** → Need(사건 발생)·Take(대응 비용)·Return(복귀)·Change(질서 변화) 강조. 알림·복구·재발 방지 시나리오에 비중.

이 매핑은 L3 챕터 가지치기에 직접 적용된다 — 분류된 축의 단계는 챕터로 살리고, 그렇지 않은 단계는 한 챕터로 합치거나 생략 가능.

## 안티패턴

- **모든 축에 똑같이 비중 두기** — "다 중요해요"는 분류가 아니다. MICE의 가치는 *비중 차이*를 강제한다는 점. 1순위·2순위까지만 살리고 나머지는 명시적으로 약화한다.
- **학술 용어 그대로 노출** — "이건 Milieu 서사예요"라고 사용자에게 말하면 인지 부하. "이건 *낯선 환경에 적응*하는 이야기네요"로 평문화한다.
- **분류 후 단계 강조 반영을 빼먹음** — Character형으로 분류해놓고 모든 단계를 똑같이 인터뷰하면 분류가 무의미. 반드시 분류 → 단계 비중 조정 → 인터뷰 깊이 차등화로 이어진다.
- **하이브리드 분류 거부** — 많은 앱이 두 축의 명확한 혼합이다(예: 건강 앱 = Character + Inquiry). 강제 단일 분류 대신 가중치 분배(예: C 60 / I 40)를 허용한다.
- **MICE를 사용자에게 "분류 시험"처럼 들이댐** — 사용자는 자기 앱을 분류하러 온 게 아니다. "어느 쪽에 *가까워요*?"·"어느 쪽 *느낌*이에요?"로 부드럽게 물어 자기 직관을 끌어낸다.

## Layer 1 메타 자세와의 결합 메모

- **Clean Language와의 결합** — 4축 라벨을 사용자가 받아들이고 자기 단어로 풀어 답할 때, 그 *자기 단어*를 그대로 받아 산출물에 기록한다. 사용자가 "환경 적응"이 아니라 "이 동네에 익숙해지기"라고 했다면, 그 표현 그대로 둔다("적응"·"익숙해지기"·"familiarize" 다 다른 단어다).
- **System 1/2 이중 캡처** — "*처음* 떠오른 분류는?" (System 1 직관 — 이게 보통 정답에 가깝다) → "30초 더 곱씹으면 다른 후보는?" (System 2 — 종종 2순위 축 발견). 둘 다 기록.
- **OARS의 Reflect** — 분류 후 한 문장 요약 ("그러니까 이건 *자기 변화*가 주축이고 *환경 적응*이 부축인 이야기네요. 맞나요?")으로 자기 정정 기회 제공.
- **Yes-And** — 사용자가 "이건 좀 다 섞인 거 같아요"라고 하면 부정하지 않는다. "맞아요. 그러면 *어느 축이 가장 강하고*, 어떤 축이 두 번째일까요?"로 비중 분배로 돌린다.
- **Bloom 게이지** — MICE는 "분석"·"평가" 층위 작업. 사용자가 "기억" 층위(과거 사례 회상)에서 그치면 "그 사례 전체를 *통틀어* 어느 축이 가장 강했나요?"로 한 단계 올린다.
